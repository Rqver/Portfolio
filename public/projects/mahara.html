Mahara

Persistent long-term memory for Claude.

About Mahara

Mahara is a persistent long-term memory system for Claude. It's a personal wiki of prose dossier pages; one per person, project, organisation, place, piece of infrastructure, and preference in my life, plus an append-only log of raw observations, all exposed to Claude over MCP.

Claude is instructed to recall from Mahara before discussing anything in my life, and to capture into it liberally. Whenever a conversation reveals something worth knowing later; a decision, a preference, a new person or project, Claude records it as an observation immediately, and a scheduled lint agent later folds observations into the right pages and dedupes.

Pages are wikilinked to each other aggressively. Fetching a page returns the page itself, stubs of every page it links to and every page that links back, and the most recent observations about it.

There's also a web panel for me: an index of every page, a feed of recent observations, search, and editing, so I can see and correct everything Claude remembers.

Why

Every conversation with Claude starts from a blank context. I use Claude across a lot of interfaces; Claude Code in the terminal, claude.ai, my phone, and I was re-explaining who people were, what projects existed, and what state everything was in, in every single conversation. Mahara gives all surfaces of Claude one shared, persistent memory.

It also compounds: the more I use Claude, the more it captures, and the better every future conversation gets.

Tech Stack

  • Written in TypeScript using Deno as a runtime.
  • The frontend is built using Tailwind CSS.
  • SQLite is used as a database.
  • The MCP server is built using the official MCP TypeScript SDK.
  • Deployed (and backed up) by Arataki.
